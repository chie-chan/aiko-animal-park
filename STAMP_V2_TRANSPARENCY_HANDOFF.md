これまで「透過はユーザーが各自で（Canva/公式アプリ）」という設計だったが、
**PC版に「白背景の自動透過」を内蔵**し、あわせて使い勝手（中央寄せ・透過確認の背景切替）と

## 2. 変更ファイル一覧

### `src/stamp-v2-split.ts`（ロジック追加）
- **`makeImageTransparent(src, opts?)` 新規**：白背景の自動透過。
  - 方式＝**縁フラッドフィル**：画像の縁から4近傍でつながった「明るく無彩色（背景候補）」ピクセルだけを α=0 にする。
    内側の白（目のハイライト・白文字・白マグ等）は border 非連結なので残る。
  - 候補判定：`max(R,G,B) >= brightMin && (max-min) <= satTol`。
  - **既定値を 222/32 → 250/8 に変更**（＝ほぼ純白だけ抜く）。理由：ゆるめだと淡い色・白い毛まで誤抜きするため。
    平面・白背景・はっきり輪郭の絵（AIスタンプ）に最適化。`opts` で上書き可。
- **`centerImageContent(src, alphaThreshold=8)` 新規**：透過後のαで被写体BBoxを取り、画像中央へ平行移動した dataURL を返す。中身が画像いっぱい（不透明画像）なら no-op。
- いずれも Canvas 2D（`getImageData`/`putImageData`）。dataURL前提なので CORS タリントなし。

### `src/Step2Splitter.tsx`（Step1 UI・状態）
- **白背景の自動透過トグル**：`bgTransparent`（既定ON）。`rawSrc`（透過前の元画像）を保持し、トグル切替で `makeImageTransparent` を掛け直す。`processing` 中は「処理中…」表示。
  - `handleFile` / `loadSample` を `applyUploadedSrc()` 経由に変更（アップ時に自動透過）。
  - トグルUIはドロップ画面＋編集画面の両方（`renderTransparentToggle`）。
- **分割プレビューの背景切替**：`liveBg`（市松/白/黒/桃/青、`LIVE_BGS`）。`.v2-live-cell` と拡大モーダルにインラインで背景適用。透過の見やすさ用。
- 文言更新：ドロップ画面の説明・サンプル注記・冒頭コメントを「白背景でもOK／自動透過」に修正。

### `src/StampToolV2.tsx`（親・フロー・文言）
- `centerImageContent` を import。
- **`handleNext()` 新規**：Step1→Step2 へ進む時に `splitCells` を全て `centerImageContent` で中央寄せ→`setSplitCells`、`cellOffsets` をリセット。`centering` 中はボタンを「中央寄せ中…」に。フッター「次へ」を `handleNext` に接続。
- **文言の整合修正（自動透過に合わせて）**：MASCOTヒント／STARTスポットライト／注意モーダル「位置づけ」「透過処理について」／使い方ガイドの流れ説明／デザインルームのコーチSTEP6／デザインルームの「公式アプリで透過しよう」ボックス。
  - 「PC＝工房が自動透過」に統一。スマホ誘導文（`/stamp-mobile`＝公式アプリ）のみ意図的に残置。

### `src/stamp-v2-frames.ts`（プロンプト：白フチなし化）
- **色シェイプ系14フレームの「外周は白フチで明確に切れる」を削除**し、
  「白フチ（白い縁取り）は付けない。色が一番外側まで来て、その外は純白の背景」に変更。
  対象＝シール風/消印/チケット/キャンディラッパー/ドーナツ/クッキー型/果物/ジャム瓶/缶バッジ/ブラウン管TV/窓枠/鏡/海/リース。
- **白が design そのものの3つ（ポラロイド／切手シール／マスキングテープ）は白を保持**（消すとデザインが破綻するため）。
- 文字の白フチ（`COMMON_TEXT_INSTRUCTIONS` の可読性用の縁取り）は**残置**＝シール枠の白フチとは別物。
- ※**プロンプト文の変更**なので、効くのは「次に生成する画像」。既存画像は不変。

`npm run dev`（localhost:5173/stamp-v2）で確認済み：
- 白背景シート→アップロードで自動透過、内側の白は保持、トグルOFFで元の白に戻る。
- しきい値250/8で、Python検証版（縁フラッドフィル）と同等のクリーンさを確認（毛抜けなし）。
- 背景切替（黒）で透過が明瞭に。

## 4. レビューで見てほしい点
1. **しきい値250/8の妥当性**：純白背景前提。背景がオフホワイト(～245)だと抜けず薄モヤが残る可能性。実運用の生成画像分布で問題ないか。
2. **`makeImageTransparent` の計算量**：シート全体（最大~1254²）に対し border BFS。`stack=Int32Array(n)` で n=画素数。大きい画像での実行時間・メモリ。
3. **`centerImageContent` を全セルに直列 await（`Promise.all`）**：9〜16枚で問題ない想定だが、巨大セル時のUIブロック有無。
4. **中央寄せ後に `cellOffsets` リセット**：Step2で手動調整→戻る→次へ で調整が消える仕様。意図どおりか（再中央寄せ＝offset0が正しいという判断）。
5. **白フチ削除の3例外（ポラロイド/切手/マステ）**の方針が妥当か。全フレーム白なし統一にすべきか。
6. **howto-2.png のピクセル編集**：フォントが原本（丸ゴシック）と完全一致ではない（meiryo bold 代替）。許容範囲か、画像再生成すべきか。
7. **モバイル版未変更**：PC＝自動透過／スマホ＝公式アプリ、の二本立てで矛盾がないか最終確認。
